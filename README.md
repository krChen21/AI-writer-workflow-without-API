# 一种无需调用API的AI工作流框架

AI 循环对谈 & 自动长时分析工具：基于浏览器自动化（Playwright）的多模型对话工具，目前支持 **DeepSeek** 和 **通义千问**，提供双模型“创作者+评价者”交替循环、单一模型持续续写两种模式，可自动导出 Word 文档。

#### `后续将接入更多的免费ai模型，并开发图像模块`

## 功能特点

- **双模型循环对话**：一个模型主创，另一个模型评价，交替优化内容
- **单一模型持续续写**：同一模型连续多轮生成，保持风格与连贯性
- **可自定义角色前缀**：灵活设定每轮评价或修改指令
- **小说风格预设**：支持分类（男频/女频/通用）、类型、文风选择
- **自动总结与导出**：结束时可生成总结文章，对话流水及总结均可导出为 `.docx` 文件
- **图形界面操作**：基于 Tkinter，无需命令行复杂参数

## 所需依赖

- Python 3.8 或更高版本
- Playwright
- python-docx
- Tkinter

## 安装步骤
1.安装所需依赖包：
```bash
# 安装依赖包
pip install playwright python-docx tkinter
```
2.开启远程调试端口：
新建一个edge浏览器快捷键→右键→属性→快捷方式→目标→在已有路径的后面添加：
```bash
--remote-debugging-port=9222 --user-data-dir="C:\playwright_edge_data"
```
3.登录 AI 网站并保持标签页打开：
在启动的 Chrome 中分别访问并登录：
```bash
DeepSeek：https://chat.deepseek.com
通义千问：https://tongyi.aliyun.com
```
务必让这两个网站各保持一个打开的标签页（不必是激活状态，但不能关闭）
## 快速开始
```bash
python main_gui.py
```

## 开始界面介绍
### 一、双模型循环对话模式（推荐）
在左侧「模型角色配置」中：

·选择主创作者（先发言）和评价者（后发言），例如 DeepSeek 为主、通义千问为评。

·填写「初始提问」，例如小说开头或技术方案要求。

·（可选）展开「小说风格预设」选择分类、类型及文风（最多 3 种）。

·在「主循环控制」中设置循环轮数（如 3 轮）。

·点击 「开始主循环」。

·观察右侧对话逐步生成，结束后自动进行总结，并弹出 Word 文件保存路径。

### 二、单一模型持续循环模式
点击左上角「单一模型持续循环模式」切换。

·选择模型、填写初始提问和中间提示词（第 2 轮及之后每轮发送）。

·设置循环轮数和端口。

·点击 「开始单一模型循环」。

### 三、继续创作（双模型模式下）
在主循环结束后，或需要额外扩写时：

·在「继续创作」区域填写起始提示词（例如“请从上一章结尾继续…”）。

·设定轮数，点击 「开始继续」。

·此时将基于当前历史对话（主循环的历史会自动继承）进行额外轮次的交替生成。


### 四、导出记录
单一模型循环或继续创作完成后，可点击对应的「导出记录」按钮保存 Word 文档。

双模型主循环结束后会自动导出两份文档（对话流水 + 总结文章）。

## 项目文件说明

### 核心模块（必须）

#### `main_gui.py`
**主程序入口**，负责创建 Tkinter 图形界面、管理用户交互、协调各模块调用。实现了三种运行模式：
- 双模型循环对话（创作者+评价者）
- 单一模型持续循环
- 双模型下的“继续创作”功能

并包含所有按钮、文本框、日志输出等界面逻辑。

#### `browser_ai.py`
**AI 交互核心**，封装了与 DeepSeek 和通义千问网页的自动化通信逻辑。
- 使用 Playwright 连接已开启调试端口的 Chrome 浏览器，定位 AI 网站的输入框、发送按钮，提取回复内容。
- 注册模型调用函数（`ask_deepseek`， `ask_qianwen`），提供统一的 `ask_model` 接口供 GUI 调用。
- 若需增加新模型（如 ChatGPT、Claude），在此文件中实现对应函数并用 `@register_model` 装饰器注册即可。

#### `Log_Export.py`
**工具函数模块**，提供：
- `update_status`：更新界面状态栏和系统日志。
- `update_output`：将 AI 回复追加到右侧对话流水区域，并记录到历史列表。
- `export_to_word`：将文本列表导出为 Word 文档（`.docx`）。
- `export_history`：封装导出操作的弹窗和文件命名。

#### `config.py`
**主配置文件**，所有可自定义的参数集中在此：
- 浏览器端口、超时时间、稳定检测次数
- 可用模型列表
- 各类默认提示词（初始提问、角色前缀、总结提示、继续创作提示）
- 小说风格预设（分类、类型映射、文风列表及选择限制）

> **实际运行时读取的是 `config.py`**，你可以根据需要修改它。

### 配置示例文件（供参考或替换）

`config_demo1.py` ~ `config_demo5.py` 是不同应用场景的配置示例，展示如何通过修改 `config.py` 来改变整个工具的行为。它们内容结构与 `config.py` 完全相同，仅预设的提示词和风格选项不同。你可以将任意一个示例文件的内容覆盖到 `config.py` 中，快速切换场景。

| 文件 | 场景 |
|------|------|
| `config_demo1.py` | 线上营销方案 |
| `config_demo2.py` | 学术论文 |
| `config_demo3.py` | 产品需求文档（PRD） |
| `config_demo4.py` | 技术方案 |
| `config_demo5.py` | 短视频脚本 |
